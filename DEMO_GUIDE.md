# 快速评估功能演示指南

## 🎯 快速测试（5分钟）

### 方法1：使用示例文件（推荐）

这是最快的测试方法，无需使用DeepSeek AI。

#### 步骤：

1. **启动应用**
   ```bash
   npm run dev
   ```

2. **填写项目信息**（可选，但建议填写）
   - 项目名称：电商平台
   - 项目描述：在线购物系统
   - 行业应用：电子商务
   - 交付端：勾选Web端、H5

3. **打开示例文件**
   - 打开项目根目录的 `example-deepseek-response.json`
   - 复制全部内容（Ctrl+A / Cmd+A，然后 Ctrl+C / Cmd+C）

4. **点击快速评估**
   - 点击左侧功能树顶部的"快速评估"按钮（⚡图标）

5. **跳过等待**
   - 对话框打开后，点击底部的"跳过等待"按钮

6. **粘贴并导入**
   - 在文本框中粘贴刚才复制的JSON内容
   - 点击"导入"按钮

7. **查看结果**
   - 左侧功能树会显示5个需求模块
   - 展开可以看到详细的功能清单
   - 底部统计栏会更新数据

✅ **预期结果：**
- 成功导入5个需求模块
- 共21个功能节点
- 可以在功能树中看到完整的层级结构

### 方法2：使用DeepSeek AI（完整流程）

这是完整的使用流程，可以体验AI生成的完整过程。

#### 步骤：

1. **启动应用**
   ```bash
   npm run dev
   ```

2. **填写项目信息**（必填）
   
   示例1 - 教育平台：
   ```
   项目名称：智慧课堂
   项目描述：面向K12的在线教育平台，支持直播授课、作业管理、学习报告
   行业应用：教育
   交付端：Web端、Android端、IOS端、小程序
   ```

   示例2 - 医疗系统：
   ```
   项目名称：智能问诊系统
   项目描述：在线医疗咨询平台，支持预约挂号、视频问诊、电子处方
   行业应用：医疗
   交付端：Web端、H5、小程序
   ```

3. **点击快速评估**
   - 点击左侧的"快速评估"按钮（⚡）
   - 系统会自动复制提示词
   - 显示"已复制"提示

4. **等待跳转**
   - 3秒倒计时
   - 自动在新标签页打开DeepSeek
   - （或点击"跳过等待"立即进入下一步）

5. **使用DeepSeek**
   - 在DeepSeek聊天界面粘贴提示词（Ctrl+V / Cmd+V）
   - 发送消息
   - 等待AI生成结果（通常10-30秒）

6. **复制AI结果**
   
   **重要：只复制JSON部分**
   
   找到类似这样的内容：
   ```json
   [
     {
       "name": "用户管理",
       "complexity": "中",
       ...
     }
   ]
   ```
   
   复制整个JSON数组（包括最外层的 `[` 和 `]`）

7. **返回系统导入**
   - 返回应用的对话框
   - 在输入框中粘贴AI生成的JSON
   - 点击"导入"按钮

8. **查看结果**
   - 左侧功能树显示导入的模块
   - 底部统计数据更新
   - 查看成功提示消息

## 🐛 常见问题排查

### Q1: 提示词没有自动复制？

**检查：**
- 浏览器是否允许剪贴板访问
- 尝试手动点击"复制"按钮
- 查看是否有浏览器安全提示

**解决：**
```
1. Chrome: 设置 → 隐私和安全性 → 网站设置 → 剪贴板
2. 允许该网站访问剪贴板
3. 刷新页面重试
```

### Q2: DeepSeek没有自动打开？

**检查：**
- 浏览器是否阻止了弹窗
- 查看地址栏是否有弹窗阻止图标

**解决：**
```
1. 点击浏览器地址栏的弹窗阻止图标
2. 选择"始终允许弹窗"
3. 或点击对话框底部的"重新打开 DeepSeek"按钮
```

### Q3: 导入时提示解析失败？

**检查：**
- 是否复制了完整的JSON
- JSON格式是否正确
- 是否包含了多余的文字说明

**解决：**
```
1. 只复制 [...] 部分
2. 可以用在线JSON验证工具检查格式
3. 确保复制时没有遗漏开头的 [ 或结尾的 ]
```

### Q4: DeepSeek没有返回JSON格式？

**追问：**
```
"请将上述内容以JSON数组格式输出，格式如下：
[
  {
    "name": "模块名",
    "complexity": "中",
    "priority": "高",
    "isImportant": true,
    "remark": "说明",
    "children": [...]
  }
]"
```

## 💡 使用技巧

### 技巧1：项目信息越详细越好
```
❌ 不好：
项目名称：系统
项目描述：管理系统

✅ 好：
项目名称：企业资产管理系统
项目描述：用于企业固定资产全生命周期管理，包括资产采购、入库、领用、调拨、维护、报废等环节，支持条码扫描和资产盘点
```

### 技巧2：明确行业特点
```
行业应用：金融 → AI会考虑合规、安全等特殊需求
行业应用：教育 → AI会考虑课程、学生等教育特色
行业应用：医疗 → AI会考虑患者、处方等医疗场景
```

### 技巧3：多次使用，逐步细化
```
第一次：生成顶层模块框架
第二次：针对核心模块深入展开
第三次：补充特殊需求
手动：添加项目特有功能
```

### 技巧4：导入后调整
```
导入后可以：
- 重命名模块
- 调整复杂度和优先级
- 添加/删除节点
- 修改详细说明
- 调整层级结构
```
